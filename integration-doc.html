DPD / Topsort Integration

Sponsored Banners — Integration Guide & Reference

1. Overview

This document demonstrates how DPD integrates Topsort's Auction & Events API to serve sponsored banners on the DPD website. The integration consists of three steps:

Auction
→
Render Banner
→
Impression
→
Click
StepEndpointWhen
AuctionPOST /v2/auctionsOn page load, before rendering the banner slot
ImpressionPOST /v2/eventsWhen 50%+ of the banner is visible for 1 continuous second (IAB standard)
ClickPOST /v2/eventsWhen the user clicks the banner

2. Auction Request

A single API call returns the winning banner creative for a given placement. The slotId identifies the placement, and opaqueUserId ties the auction to the user session for frequency capping and attribution.

Request

POST https://api.topsort.com/v2/auctions

{
  "auctions": [
    {
      "type": "banners",
      "slots": 1,
      "slotId": "home",
      "opaqueUserId": "557ca396-6bdc-44db-b5d8-08e8c70e81bf"
    }
  ]
}

Response

{
  "results": [
    {
      "resultType": "banners",
      "winners": [
        {
          "rank": 1,
          "type": "vendor",
          "id": "vendor1",
          "resolvedBidId": "ZyahrwoQBp3lSX0N...",
          "asset": [
            {
              "url": "https://topsortassets.com/asset_01kp654znz...json",
              "content": {
                "imageURL": "https://topsortassets.com/asset_01kp654qs1...png",
                "redirectURL": "https://www.dpd.com/",
                "altText": "Valentine's Day Sale – Up to 20% Off"
              }
            }
          ]
        }
      ],
      "error": false
    }
  ]
}

The resolvedBidId is the key value to retain — it links the auction to subsequent impression and click events.

3. Fallback Banners

When an auction is scoped to product IDs that have no active campaigns, Topsort returns fallback assets. These use a simpler response shape — a direct url on the asset instead of a content object.

Request

POST https://api.topsort.com/v2/auctions

{
  "auctions": [
    {
      "type": "banners",
      "slots": 2,
      "slotId": "home",
      "opaqueUserId": "557ca396-6bdc-44db-b5d8-08e8c70e81bf",
      "products": {
        "ids": ["non-existing"]
      }
    }
  ]
}

Response

{
  "results": [
    {
      "resultType": "banners",
      "winners": [
        {
          "rank": 1,
          "type": "url",
          "id": "https://www.dpd.com/",
          "resolvedBidId": "xnXkqgoQBp3lSZML...",
          "asset": [
            {
              "url": "https://assets.hosted.topsort.com/.../FallbackBanner2.png"
            }
          ]
        }
      ],
      "error": false
    }
  ]
}

The rendering logic handles both shapes: if asset[0].content exists, use imageURL / redirectURL; otherwise fall back to asset[0].url.

4. Event Reporting

Impressions and clicks are reported to the same endpoint with the resolvedBidId from the auction. Each event requires a unique id (UUID v4) and an ISO 8601 occurredAt timestamp.

Impression (IAB-compliant)

Fires when 50%+ of the banner is visible in the viewport for 1 continuous second, implemented via IntersectionObserver. 

POST https://api.topsort.com/v2/events

{
  "impressions": [
    {
      "id": "2770ca99-0ca3-47c9-8bce-0a1a682dce3c",
      "occurredAt": "2026-04-14T15:03:44.264Z",
      "opaqueUserId": "557ca396-6bdc-44db-b5d8-08e8c70e81bf",
      "resolvedBidId": "ZyahrwoQBp3lSX0N..."
    }
  ]
}

Click

6. Key Integration Points

FieldPurpose
opaqueUserIdTies auction, impressions, and clicks to the same user session. Can be a hashed anonymous ID or the authenticated user ID.
resolvedBidIdReturned by the auction. Must be forwarded to both impression and click events for proper attribution.
slotIdIdentifies the placement on the page (e.g. "home"). Maps to the slot configured in Topsort.
idUnique UUID per event call. Ensures idempotency and deduplication.
